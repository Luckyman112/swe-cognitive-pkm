# Cognitive PKM (Personal Knowledge Management)

##  Goal
This repository serves as my structured cognitive environment for the Software Engineering course. It optimizes for decision traceability, versioned learning, and capturing insights from transitioning to AI-Native development.

##  Folders Structure
- `/concepts` — Theoretical notes and definitions (e.g., OOP principles, Design Patterns).
- `/decisions` — Architecture Decision Records (ADRs) explaining *why* certain tools or patterns were chosen for the Task Manager CLI.
- `/experiments` — Logs of AI code generation tests (BDD, Pure Functions).
- `/logs` — Weekly reflections and lecture insights.

##  Rules of Engagement
1. Every major lecture insight goes into `/logs`.
2. Architectural choices for the Product Repo must be traced in `/decisions`.
3. AI interactions must be strictly documented to evaluate the Engineering Harness.
